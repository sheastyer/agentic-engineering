Cross-cutting

Budget gate human gate7d
After every stage, accumulated spend is checked against the per-workflow ceiling ($3 feature · $0.50 bug). Crossing it trips a budget_override signal: approve once → continue (won’t re-trip) · decline/timeout → OVER_BUDGET (terminal). Cost is dollar-denominated from real response.usage × tier pricing.
Invariants: deterministic workflow code (no LLM/IO/clock/randomness) · human gates are signals with timeouts · every agent↔agent loop is bounded · deploy is always human-gated · coding agents run sandboxed. See CLAUDE.md §9.
